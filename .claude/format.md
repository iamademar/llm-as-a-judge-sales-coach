# CLAUDE RULES — OUTPUT FORMATTING

## Purpose
Ensures consistent technical output formatting.

## Mandatory Rules
- Use Markdown headings.
- Use numbered steps for procedures.
- Use tables for comparisons.
- Always annotate code blocks with language.

## Prohibited Actions
- Do NOT produce monolithic unstructured output.
- Do NOT mix languages in a single code block.

## Self-Verification Checklist
- ✅ Is formatting clear?
- ✅ Are commands isolated?
- ✅ Are examples labeled?
